If you're learning AI, RAG, embeddings, and LLMs, here is a comprehensive list of **LLM (Large Language Model) terminologies** grouped by category.

---

# 1. Core LLM Concepts

| Term               | Description                                                          |
| ------------------ | -------------------------------------------------------------------- |
| LLM                | Large Language Model trained on massive text datasets.               |
| Token              | Smallest unit of text processed by an LLM.                           |
| Context Window     | Maximum number of tokens the model can remember in one conversation. |
| Prompt             | Input given to the model.                                            |
| Completion         | Output generated by the model.                                       |
| System Prompt      | Instructions that define the AI's behavior.                          |
| User Prompt        | Message sent by the user.                                            |
| Assistant Response | Model-generated reply.                                               |
| Temperature        | Controls randomness (0 = deterministic, 1+ = creative).              |
| Top-p              | Sampling method limiting token choices by cumulative probability.    |
| Top-k              | Restricts selection to the top K likely tokens.                      |
| Max Tokens         | Maximum length of generated output.                                  |
| Stop Sequence      | Text that tells the model when to stop generating.                   |

---

# 2. Tokenization

| Term                     | Description                          |
| ------------------------ | ------------------------------------ |
| Tokenizer                | Converts text into tokens.           |
| Encoding                 | Converting text into token IDs.      |
| Decoding                 | Converting token IDs back into text. |
| Vocabulary               | Set of all known tokens.             |
| Byte Pair Encoding (BPE) | Popular tokenization algorithm.      |
| SentencePiece            | Google's tokenizer.                  |
| WordPiece                | Used by many transformer models.     |

---

# 3. Transformer Architecture

| Term                 | Description                                              |
| -------------------- | -------------------------------------------------------- |
| Transformer          | Neural network architecture used by modern LLMs.         |
| Attention            | Mechanism allowing the model to focus on relevant words. |
| Self-Attention       | Words attend to other words in the same sentence.        |
| Multi-Head Attention | Multiple attention mechanisms running simultaneously.    |
| Encoder              | Reads input (used in models like BERT).                  |
| Decoder              | Generates output (used in GPT models).                   |
| Feed Forward Network | Neural layer after attention.                            |
| Positional Encoding  | Gives the model information about word order.            |
| Hidden State         | Internal representation of text.                         |

---

# 4. Embeddings

| Term               | Description                                         |
| ------------------ | --------------------------------------------------- |
| Embedding          | Numerical vector representation of text.            |
| Vector             | List of numbers representing meaning.               |
| Embedding Model    | Model that converts text into vectors.              |
| Similarity Search  | Finds similar vectors.                              |
| Cosine Similarity  | Common method to compare vectors.                   |
| Euclidean Distance | Distance between vectors.                           |
| Semantic Search    | Search based on meaning rather than exact keywords. |

---

# 5. Vector Database

| Term            | Description                                           |
| --------------- | ----------------------------------------------------- |
| Vector Database | Stores embeddings for similarity search.              |
| Index           | Data structure for fast retrieval.                    |
| ANN             | Approximate Nearest Neighbor search.                  |
| HNSW            | Hierarchical graph index for efficient vector search. |
| IVF             | Inverted File Index.                                  |
| Metadata        | Additional information stored with vectors.           |
| Hybrid Search   | Combines keyword and vector search.                   |

Popular vector databases include Pinecone, Milvus, Weaviate, and Qdrant.

---

# 6. RAG (Retrieval-Augmented Generation)

| Term          | Description                                      |
| ------------- | ------------------------------------------------ |
| RAG           | Combines document retrieval with LLM generation. |
| Retriever     | Finds relevant documents.                        |
| Generator     | Produces the final answer.                       |
| Chunking      | Splitting documents into smaller pieces.         |
| Chunk Size    | Number of words or tokens per chunk.             |
| Chunk Overlap | Shared text between adjacent chunks.             |
| Retrieval     | Fetching relevant information.                   |
| Re-ranking    | Reordering retrieved results by relevance.       |
| Grounding     | Answering using trusted external data.           |
| Hallucination | Confident but incorrect model output.            |
