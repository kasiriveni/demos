Agents

Build powerful custom agents for specialized development tasks

Basics

✓ What is an Agent?

An Agent is an AI-powered assistant that can understand context, execute tasks autonomously, and interact with your codebase. Unlike simple chatbots, agents can plan multi-step workflows, use tools, and make decisions based on the project context.

Key Features:

  • Autonomous task execution
  • Context-aware decision making
  • Tool and API integration
  • Multi-step planning capabilities
✓ Agent vs Chat

Chat: Simple Q&A interface with immediate responses. Limited to single-turn interactions.

Agent: Can execute complex, multi-step tasks autonomously. Maintains context across interactions, can use tools, make plans, and work independently toward goals.

Think of chat as a conversation, while an agent is a collaborator that can actually do work for you.

✓ Agent Lifecycle

The agent lifecycle consists of several phases:

  1. Initialization: Agent loads configuration and instructions
  2. Context Gathering: Analyzes workspace, files, and dependencies
  3. Planning: Creates a strategy to accomplish the task
  4. Execution: Carries out the plan using available tools
  5. Verification: Checks results and iterates if needed
  6. Completion: Reports results and updates state
✓ Agent Persona

Agent Persona defines the personality, communication style, and behavior patterns of your agent. This includes:

  • Expertise Level: Junior dev, senior engineer, architect
  • Communication Style: Formal, casual, technical, educational
  • Focus Areas: Security, performance, testing, documentation
  • Decision Making: Conservative, aggressive, balanced

Example: A "Security-First Agent" would prioritize vulnerability scanning and secure coding practices in all suggestions.

✓ Agent Capabilities

Agents can perform various capabilities:

  • Code Generation: Write new code from scratch
  • Code Analysis: Review and understand existing code
  • Refactoring: Improve code structure and quality
  • Testing: Generate and run tests
  • Documentation: Create comprehensive docs
  • Debugging: Find and fix issues
  • Tool Integration: Connect to external services
  • Workflow Automation: Automate repetitive tasks

Advanced

✓ Multi-Agent Architecture

A system where multiple specialized agents work together, each handling specific aspects of development:

  • Frontend Agent: Handles UI/UX code
  • Backend Agent: Manages API and business logic
  • Database Agent: Optimizes queries and schemas
  • DevOps Agent: Handles deployment and infrastructure

Each agent operates independently but can coordinate through a master orchestrator.

✓ Agent Collaboration

Agents can work together by sharing context, results, and coordinating actions. For example:

A Code Review Agent identifies issues → Security Agent checks vulnerabilities → Testing Agent generates tests → Documentation Agent updates docs.

This creates a comprehensive development pipeline automated by AI agents.

✓ Agent Delegation

When an agent encounters a task outside its expertise, it can delegate to another specialized agent. For example, a React Agent working on a component might delegate database query optimization to a SQL Agent.

This enables specialization while maintaining flexibility.

✓ Tool Calling

Agents can invoke external tools and APIs to extend their capabilities:

  • Run terminal commands
  • Execute API requests
  • Query databases
  • Access file systems
  • Call third-party services (GitHub, Jira, etc.)

Tools are defined with schemas that specify inputs, outputs, and usage patterns.

✓ Autonomous Coding

Agents can write entire features or fix bugs without human intervention. The agent:

  1. Analyzes the requirements
  2. Plans the implementation
  3. Writes the code
  4. Runs tests
  5. Fixes any issues
  6. Creates documentation
  7. Submits for review

All while following your coding standards and best practices.

✓ Planning & Memory

Planning: Agents break down complex tasks into actionable steps, creating a roadmap before execution.

Memory: Agents maintain context across sessions:

  • Short-term: Current conversation and task context
  • Long-term: Project patterns, preferences, and historical decisions
  • Semantic: Understanding of codebase structure and relationships
✓ Context Management

Managing what information the agent has access to:

  • Workspace Context: Project files, structure, dependencies
  • Git Context: History, branches, commits
  • Environment Context: System info, installed tools
  • User Context: Preferences, patterns, conventions

Effective context management ensures relevant information without overwhelming the agent.

✓ Long-running Tasks

Agents can handle tasks that take hours or days:

  • Migrating entire codebases
  • Refactoring large applications
  • Generating comprehensive test suites
  • Building complete features

The agent can pause, resume, handle interruptions, and provide progress updates throughout the process.

Skills

Reusable capabilities that supercharge your agents

Topics to Master

What are Skills?

Skills are reusable, modular capabilities that agents can execute. They encapsulate specific knowledge and procedures for common tasks like generating APIs, running tests, or optimizing code.

Think of skills as specialized functions that your agent can call upon when needed.

Skill Structure & Files

A skill typically consists of:

  • SKILL.md: Main skill definition and instructions
  • metadata.json: Skill configuration and parameters
  • examples/: Usage examples and test cases
  • templates/: Code templates and patterns

Skills are organized in a directory structure that makes them easy to discover and use.

Inputs & Outputs

Inputs: Parameters, context, and data the skill needs to execute (e.g., file paths, configuration options, user preferences).

Outputs: Generated code, files, reports, or data structures that result from executing the skill.

Well-defined inputs and outputs make skills composable and predictable.

Skill Parameters

Parameters customize skill behavior:

  • Required: Must be provided (e.g., entity name)
  • Optional: Have defaults (e.g., output format)
  • Flags: Boolean options (e.g., include tests)

Example: generate-api --entity=User --format=REST --with-tests

Skill Chaining

Connect multiple skills together where the output of one becomes the input of another:

Generate API → Add Validation → Generate Tests → Create Docs

This creates powerful automated workflows for complex tasks.

Reusable Skills

Skills are designed to work across different projects and contexts. A "Generate CRUD" skill can work for any entity in any framework.

Benefits:

  • DRY principle for AI instructions
  • Consistent patterns across projects
  • Share skills across teams
  • Build skill libraries
Skill Versioning

Track skill versions to ensure compatibility and manage updates:

  • v1.0.0: Initial release
  • v1.1.0: New features added
  • v2.0.0: Breaking changes

Use semantic versioning to communicate changes clearly.

Domain-specific Skills

Specialized skills for particular domains:

  • DevOps: Docker, Kubernetes, Terraform
  • Database: SQL optimization, migration generation
  • Frontend: React components, responsive layouts
  • Backend: API endpoints, authentication
  • Security: Vulnerability scanning, secure patterns

Instructions

Define the AI's always-on behavior and coding standards

Instruction Types

Global Instructions

Global instructions apply across all projects and workspaces for your user account. They define your personal coding style and preferences that should always be followed.

Examples:

  • Always use TypeScript over JavaScript
  • Prefer functional programming patterns
  • Use 2 spaces for indentation
  • Write descriptive commit messages
Project Instructions

Project-specific instructions that apply only within a particular workspace or repository. These override global instructions and define project-specific standards.

Use cases:

  • Framework-specific patterns (React, Vue, Angular)
  • API conventions for this project
  • Database schema naming rules
  • Component structure guidelines
Personal Instructions

Your individual preferences that shape how the AI interacts with you, regardless of the project.

Examples:

  • Explain code changes in detail
  • Always suggest performance optimizations
  • Prioritize code readability over brevity
  • Include inline comments for complex logic

Coding Standards

Naming Conventions

Define how variables, functions, classes, and files should be named throughout your codebase.

Common conventions:

  • camelCase for variables and functions
  • PascalCase for classes and components
  • UPPER_SNAKE_CASE for constants
  • kebab-case for file names
  • Descriptive names over abbreviations
Architecture Rules

Guidelines for organizing code structure and system design patterns.

Examples:

  • Follow MVC/MVVM patterns
  • Implement clean architecture layers
  • Use dependency injection
  • Separate concerns (business logic, UI, data)
  • Apply SOLID principles
Testing Rules

Standards for writing and organizing tests across your codebase.

Guidelines:

  • Write unit tests for all business logic
  • Aim for 80%+ code coverage
  • Use descriptive test names
  • Follow AAA pattern (Arrange, Act, Assert)
  • Mock external dependencies

Quality Rules

Security Rules

Security best practices that must be followed in all code.

Requirements:

  • Never hardcode secrets or API keys
  • Sanitize all user inputs
  • Use parameterized queries (prevent SQL injection)
  • Implement proper authentication/authorization
  • Keep dependencies updated
Performance Rules

Guidelines for writing efficient, performant code.

Best practices:

  • Optimize database queries (indexes, eager loading)
  • Use caching where appropriate
  • Avoid N+1 query problems
  • Lazy load resources when possible
  • Profile and measure before optimizing
Documentation Rules

Standards for documenting code and maintaining project documentation.

Requirements:

  • Document all public APIs and functions
  • Keep README files up to date
  • Add JSDoc/TypeDoc comments
  • Document complex algorithms inline
  • Maintain architecture decision records (ADRs)

MCP Servers

Model Context Protocol - Connect to external services and data

What is MCP?

The Model Context Protocol (MCP) is one of the most important advanced topics. It enables agents to connect to external services, databases, and APIs, providing rich contextual information.

Basics

What is MCP?

The Model Context Protocol (MCP) is a standardized protocol that allows AI agents to connect with external data sources, tools, and services. It provides a unified way to extend agent capabilities beyond their built-in knowledge.

Why MCP?

MCP solves the problem of agents needing real-time, contextual information:

  • Access live data from databases and APIs
  • Connect to development tools (GitHub, Jira, CI/CD)
  • Standardized protocol across different AI models
  • Secure, controlled access to resources
MCP Architecture

MCP uses a client-server architecture:

  • Client: The AI agent that requests information
  • Server: Provides access to specific resources or capabilities
  • Protocol: Defines how messages are exchanged
  • Transport: The communication channel (stdio, HTTP, WebSocket)
Client & Server

MCP Client: Integrated into AI assistants (like GitHub Copilot), sends requests to servers.

MCP Server: Standalone service that exposes tools, resources, and prompts to clients. Can be written in any language.

Transport Layer

The transport layer handles communication between client and server. Supports multiple protocols based on deployment needs (local process, remote service, etc.).

Communication

JSON-RPC

MCP uses JSON-RPC 2.0 for message formatting - a lightweight remote procedure call protocol that encodes messages as JSON. Enables structured requests and responses with method names, parameters, and results.

stdio

Standard input/output communication - perfect for local MCP servers running as subprocesses. Fast and efficient for single-machine setups. The agent launches the server and communicates through stdin/stdout.

HTTP

Traditional HTTP request-response for remote MCP servers. Widely supported, works across networks, good for stateless operations. Can be secured with HTTPS and standard authentication.

Server-Sent Events (SSE)

One-way streaming from server to client over HTTP. Useful for real-time updates, progress notifications, and streaming responses from MCP servers without full WebSocket overhead.

WebSocket

Full-duplex bidirectional communication channel. Best for real-time, interactive MCP servers that need to push updates or maintain stateful connections. More complex but most flexible.

Tools & Resources

Tool Discovery

The process by which agents learn what capabilities an MCP server provides. Servers advertise available tools with their names, descriptions, and parameters, allowing agents to understand what they can do.

Tool Calling

When an agent invokes a tool from an MCP server, passing parameters and receiving results. The agent decides when and how to use tools based on the task at hand.

Resources

Data sources that MCP servers can expose: files, database records, API responses, documentation, etc. Resources provide context that agents can use to make informed decisions.

Prompts

Pre-configured prompt templates that MCP servers can provide to agents. These guide how agents interact with the server's capabilities and format their requests.

Context Sharing

How MCP enables agents to access and share relevant context: workspace info, user preferences, project state. This makes agents context-aware and more effective.

Build MCP Servers

Python MCP Server

Build MCP servers using Python - ideal for data science, ML integration, and rapid development. Use official MCP SDK for Python to define tools and resources.

Great for: Database queries, data analysis, file processing, API integrations

Node.js MCP Server

JavaScript/Node.js MCP servers for web services, real-time features, and JavaScript ecosystem integration. Excellent performance for I/O-heavy operations.

Great for: Web APIs, WebSocket services, npm package integration

TypeScript MCP Server

TypeScript adds type safety to Node.js MCP servers. Get compile-time error checking, better IDE support, and more maintainable code for complex servers.

Great for: Large-scale servers, team projects, type-safe API definitions
